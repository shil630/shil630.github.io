产品评审会上，老板一般会问："这次改版 ROI 有多少？"

德州牌桌上，玩家问的是另一句话："这手牌 EV 多少？"


## 这两套黑话，从计算的公式看其实**算的是一回事**：结果还没发生，这笔Bet砸下去到底值不值。

ROI，长这样：

**ROI = (收益 − 成本) / 成本**

EV，长这样：

**EV = P(赢) × 赢得的金额 − P(输) × 输掉的金额**


| 德扑 | 产品改版 |
|------|----------|
| 赢得的金额 | 改版带来的增量收益（留存、转化、客诉少了折算成钱） |
| 输掉的金额 | 改版总成本 + 翻车了要赔的（回滚、用户流失、品牌挨骂） |
| P(赢) | 改版真能达标的概率 |
| P(输) | 没达标，或者搞出副作用的概率 |
| 底池（Pot） | 你不改的话，那个烂导航会一直漏多少钱 |
| 下注额（Bet） | 这次要投进去的人周、工期、用户骂你的精神损失费 |


产品同学最容易犯的错是啥？只填 ROI 分子——"改版后转化能涨 20%哦！"——分母呢？概率呢？不好意思忘了。

这在牌桌上等于跟人说"我能赢很多！"，但不说自己跟了多少、对面 range 有多宽。对手听了都想笑。

**没有概率的收益，那不叫收益，叫许愿。**

## 来，咱们真算一笔

假设你要改导航，数字随便编的，但逻辑是真的（别较真，领会精神）：

**成本（你得下多少注）**
- 3 个工程师 × 6 周 ≈ 18 人周
- 内部成本按 8,000 元/人周算 → **144,000 元**
- 老用户被改懵了，预估月活流失 2%，一年大概亏 **80,000 元**
- 合计下注：**224,000 元**

**收益（要是成了呢）**
- 新用户 onboarding 完成率 40% → 52%，涨了 12 个点
- 每月新增 10,000 注册，多捞 1,200 个完成 onboarding 的
- 客单价 50 块 → 月多赚 60,000，一年 **720,000 元**

**概率（最难填，也最爱瞎填）**
- 乐观：60% 能成
- 基准：35% 吧，比较诚实
- 悲观：15% 彻底翻车，回滚，再亏 50,000

代入 EV：

**EV = 0.35 × 720,000 − 0.65 × 224,000 − 0.15 × 50,000**
**= 252,000 − 145,600 − 7,500 = +99,900 元**

嗯，基准情景下 EV 是正的。这手牌，**能打**。

但你要是拍脑袋把成功率写成 70% 呢？EV 立马飙到 +280,000——爽不爽？爽。靠不靠谱？大概率不靠谱。

德扑玩家管这个叫 **overestimate equity**：觉得自己胜率比实际高一截，然后莫名其妙就推大注了。产品版就是评审会上把 P(赢) 吹上天，预算批得痛快，三个月后集体沉默。

| 你瞎填的部分 | 牌桌上咋死 | 改版里咋死 |
|-------------|-----------|-----------|
| 胜率 (+10%) | 边缘牌打巨池 | 争取预算时"感觉能成" |
| 赢面 (+20%) | 以为能榨值，人家根本不跟 | 只算新增收入，存量损伤装看不见 |
| 成本 (−30%) | 隐含赔率？不存在的 | "两周搞定"，最后六周还在改 |

**ROI 报告里最大的坑，十有八九藏在概率那一格。** 你懂的。

## 啥时候改，啥时候直接 fold

牌桌上 fold 不是怂，是**"这手牌数学上亏，我不打了"**。产品里"先不改"也一样——你其实在打另一手牌：赌这个问题不会变得更糟，或者糟得没那么快。

常见就三种局面：

### 1. EV 是正的，但 ROI 看着不起眼

小底池，小下注，赢了也就赢顿外卖钱。改个按钮文案啥的，成本极低，收益也有限。能打，但别指望它拯救全年 KPI，那不现实。

这就是**小步迭代**嘛。EV 为正就值得做，复盘会上也别硬吹成"战略级变革"，怪尴尬的。

### 2. ROI 巨好看，但 EV 其实是负的

这个太常见了。视觉翻新完，老板点头说"品牌高级了"，核心指标纹丝不动，老用户骂娘骂到客服崩溃。成本两百万，收益全靠 narrative。

牌桌上这叫 **fancy play syndrome**——打得花里胡哨，账面上亏钱。我的经验是：**PPT 越精美，越要警惕你是不是在诈唬自己人。**

### 3. EV 是正的，但方差大到吓人

成功率 40%，成了赚五百万，输了亏三百万。期望值为正，可公司未必扛得住这一把梭哈。

投资你会分散仓位吧？德扑你会控制 bet sizing 吧？产品这边就该 **灰度、分期、画好回滚线**——不是说这手牌不对，是别一口气把筹码全推了，留点余地。

> 正 EV 不等于 all-in。正 EV 加上高方差，得学会控仓位，不然赢了是故事，输了是事故。

## 设计在这公式里算哪块

有人觉得设计是 ROI 表外面那层"好看的皮肤"。不是的，设计直接动两个核心变量：

**1. P(赢)——成功率**

信息架构搞清楚了，用户少迷路，成功率就上去。不是因为"更美"，是因为**人家不容易点错了**。用户找不到按钮，别扯审美，你的胜率在掉啊。

**2. 成本——下注额**

导航多一层、交互多一种花样，都是在往上加筹码。极简主义为啥有商业价值？不是设计师任性，是**下注额压下去了，赢面还没砍**。

所以设计评审别老问"你喜不喜欢"，多问两句实在的：

- 这方案让 P(赢) 涨了多少？凭啥？
- 成本涨了多少？有没有更便宜的招儿达到同样效果？

答不上来？那基本就是在用设计搞诈唬了。

## 填完这张表再开会

下次谁提改版，先甩这张表。空着？不好意思，评审会下次再说。

| 项目 | 金额/概率 | 备注 |
|------|----------|------|
| 直接成本（人周、外包、工具） | | |
| 隐性成本（迁移摩擦、机会成本） | | |
| 成功时年化收益 | | 只写一项核心指标，别贪多 |
| 失败时额外损失 | | 回滚、客诉、流失都算进去 |
| 成功概率（基准） | | 写依据：数据？访谈？还是竞品意淫？ |
| **EV（基准）** | | 代入公式，别手算心算 |
| 弃牌线 | | 啥数据一出来就停或回滚 |

EV 算出来是负的，接下来该聊的是"怎么少下注"或者"怎么先花小钱买点信息"，而不是"要不再调一版配色？"——配色救不了负 EV，真的。

**买信息**这事儿，德扑里叫 probe bet，小注探一下；产品里叫可用性测试、5% 灰度、扔个原型出去看看。花五万块把成功率从 35% 摸到 55%，有时候比直接砸两百万全量上线的 EV 高多了。划算吧？

## 别拿结果反推决策质量

说个经典的：AK 翻前 all-in，河牌被 72 反超。决策错了吗？没错，你碰上小概率了。

反过来，94 瞎跟到底，中了个顺子。决策对了吗？也不对，你只是**用错误打法赢了一回**。

改版一个德行。上线 DAU 涨了，不代表当初决策英明；指标没动，也不代表团队蠢——可能就是方差、观察窗口太短、或者你压根在量错的指标。

职业牌手复盘，看的是**当时知道啥、当时赔率多少、当时下了多少注**。产品复盘也该这么来。

光盯着结果评 ROI 呢，你会学到一堆歪理：开始迷信诈唬式改版，开始不敢做正 EV 的小迭代，开始在竞品发版后上头（tilt），跟着人家的节奏连续跟注。

别，真别。

---

说白了，产品改版和德州扑克都是：**钱有限、信息不全、结果还滞后**。挺折磨人的，但也挺公平。

老板嘴里的 ROI，牌手嘴里的 EV，投资人嘴里的期望值——同一个数，三种方言。

**把成本、收益、概率塞进同一个公式，再决定下多少注。** 视觉稿什么的，不过是让这张表更容易填平罢了。
